Тест 1. Проверка функции входа в систему:

    1) Пользователь заходит на сайт. Ожидается, что откроется главная страница.
    2) Пользователь нажимает на кнопку "Войти". Ожидается переход на страницу для ввода данных авторизации.
    3) Пользователь вводит несуществующий адрес электронной почты admin@gmail.com в поле для электронной почты. Ожидается, что поле для ввода электронной почты будет заполнено.
    4) Пользователь вводит несуществующий пароль "admin" в поле для пароля. Ожидается, что поле для ввода пароля будет заполнено.
    5) Пользователь нажимает на кнопку "Войти". Ожидается отправка формы и отображение сообщения об ошибке.
    6) Проверка наличия сообщения об ошибке авторизации. Ожидается, что сообщение об ошибке будет отображено.
    7) Пользователь закрывает сообщение об ошибке, нажимая на кнопку "крестик". Ожидается, что сообщение об ошибке исчезнет.
    8) Пользователь нажимает на логотип сайта. Ожидается переход на главную страницу.

Тест 2. Проверка функции поиска на сайте:

    1) Пользователь заходит на сайт. Ожидается, что откроется главная страница.
    2) Пользователь вводит слово "котята" в строку поиска на главной странице и нажимает клавишу Enter. Ожидается переход на страницу с результатами поиска.
    3) Проверка наличия заголовка страницы, содержащего слово "Котята". Ожидается, что такой заголовок будет присутствовать.
    4) Проверка наличия результатов поиска (изображений) на странице. Ожидается, что изображения будут отображены в результатах поиска.
    5) Пользователь нажимает кнопку сброса строки поиска. Ожидается, что строка поиска будет очищена.
    6) Пользователь нажимает кнопку "Поиск по изображению". Ожидается появление выпадающего окна с вариантами вставки изображения.
    7) Пользователь вставляет URL изображения в строку поиска в выпадающем окне. Ожидается выполнение поиска по выбранному изображению и переход на страницу с результатами поиска.
    8) Проверка наличия выбранного изображения в качестве образца на новой странице. Ожидается, что изображение будет присутствовать на новой странице.
    9) Пользователь нажимает на логотип сайта. Ожидается переход на главную страницу.

    8) Пользователь нажимает на кнопку копирования URL адреса профиля. Ожидается, что URL будет скопирован.
    9) Пользователь закрывает модальное окно. Ожидается его закрытие.
    10) Пользователь снова нажимает на кнопку "Дополнительные действия" на странице профиля автора. Ожидается появление выпадающего окна со списком возможностей.

    12) Пользователь выбирает первую причину жалобы из списка. Ожидается, что причина жалобы будет выбрана в списке.
    13) Пользователь вводит дополнительную информацию о жалобе. Ожидается, что поле для ввода информации о жалобе будет заполнено.
    14) Пользователь закрывает модальное окно репорта. Ожидается его закрытие.
    15) Пользователь кликает на кнопку "Нравится" на странице профиля автора. Ожидается переход на страницу с изображениями, которые автор добавил в избранное.
    16) Пользователь кликает на кнопку "Коллекции" на странице профиля автора. Ожидается переход на страницу с коллекциями автора.
    17) Пользователь кликает на первую коллекцию в списке коллекций автора. Ожидается переход на страницу коллекции.
    18) Проверка наличия изображений в коллекции. Ожидается наличие изображений в коллекции.
    19) Пользователь нажимает на логотип сайта. Ожидается переход на главную страницу.
